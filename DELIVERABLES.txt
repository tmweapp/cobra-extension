API ENDPOINT REFERENCE
================================================================================

BASE URL: https://your-vercel.vercel.app/api/team-auth

PUBLIC ENDPOINTS:
─────────────────
POST ?action=register
  - Body: { email, password, name }
  - Returns: { user: { id, email, name, role } }

POST ?action=login
  - Body: { email, password }
  - Returns: { token, user, team?, shared_keys? }

ADMIN-ONLY ENDPOINTS (Require JWT Token):
─────────────────────────────────────────
POST ?action=admin-login
  - Body: { email, password }
  - Returns: { token, user, admin_team }

POST ?action=admin-add-member
  - Body: { token, email, role, token_limit?, date_limit? }
  - Returns: { id, email, name, role, token_limit, date_limit }

POST ?action=admin-remove-member
  - Body: { token, user_id }
  - Returns: { message: "Member removed" }

GET ?action=admin-list-members&token=<jwt>
  - Returns: { team_id, team_name, max_tokens_per_user, members[] }

POST ?action=admin-set-keys
  - Body: { token, openai_key, anthropic_key, gemini_key, groq_key, eleven_key }
  - Returns: { message: "Keys saved" }

GET ?action=admin-get-keys&token=<jwt>
  - Returns: { openai_key, anthropic_key, gemini_key, groq_key, eleven_key }

AUTHENTICATED ENDPOINTS (Require JWT Token):
──────────────────────────────────────────
GET ?action=check-access&token=<jwt>
  - Returns: { valid: bool, user, team?, shared_keys?, reason? }

POST ?action=track-usage
  - Body: { token, provider, tokens_used }
  - Returns: { message: "Usage tracked" }

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

1. "Invalid credentials" on login
   → Check user exists in cobra_users table
   → Verify password hash matches
   → Check user hasn't been deleted

2. "Token limit reached" error
   → Admin increases limit: admin-set-keys → update token_limit
   → Check team_settings.max_tokens_per_user
   → Review member's tokens_used value

3. "Decryption error" with API keys
   → Verify ENCRYPTION_KEY length (must be 32 chars)
   → Check ENCRYPTION_KEY hasn't changed between saves
   → Re-enter keys if corrupt

4. "Admin access required" for team member
   → Verify user role = 'admin' in cobra_users
   → Check team_settings.admin_user_id matches user ID
   → Create team_settings if missing

5. Shared keys not loading for team member
   → Verify team membership in cobra_team_members
   → Check active = true for member
   → Verify shared keys are actually saved
   → Check date_limit hasn't passed (if invited)
